1. What is the difference between map, filter, foreach and find?

Ans: forEach() applies a function to each element of a collection, but does not return a new collection. Instead, it is used for performing some operation on each element in the collection, such as printing or updating a variable. In forEach() we need to execute a function for each individual element in an array. Good practice is that we should use .forEach() when we can not use other array methods to accomplish your goal.

.map() applies a function to each element of a collection and returns a new collection containing the result. For example, map can be used to convert an array of integers to an array of their squares. We can use .map() when we want to transform elements in an array.

.filter() applies a function to each element of a collection and returns a new collection containing only those elements for which the function returns true. We should use .filter() when we want to select a subset of multiple elements from an array.

.find() applies a function to each element of a collection and returns the first element for which the function returns true. If no element satisfies the condition, it returns undefined. We can use .find() When we want to select a single element from an array.

In short the basic defference is, map and filter return new collections, foreach performs an operation on each element without returning anything, and find returns the first element that satisfies a condition.

2. what is the difference between local storage and session storage and cookies?

Local Storage Local storage is a type of storage that allows web applications to store data on a user's browser with no expiration date. It is a web storage method that helps us store data on the client’s computer in the form of key/value pairs in a web browser. Local storage provides a way for web applications to persist data even after the browser is closed, and is typically used to store user preferences or application data. Local storage is accessible from the client-side only.

Session Storage Session storage is a type of storage that allows web applications to store data on a user's browser for the duration of a session (i.e., until the user closes the browser tab or window). It is very similar to the local storage. Still, the main difference lies in the lifespan as it persists in the browser until its current tab is on. Once you close the tab or terminate it, the data on session storage also gets lost. Session storage is accessible from the client-side only.

Cookies Cookies are small text files that a website stores on a user's browser. The only option that was available before HTML5 was introduced was cookies. They are typically used to store user preferences or session information such as login status, shopping cart contents, or language settings. It help us store the client-side data to enable a personalized experience for the website’s users. Cookies have an expiration date, and can be either persistent (saved on the hard drive) or session-based (deleted when the user closes the browser). Cookies are accessible from both the client and the server-side.

In summary, cookies are used to store user preferences and session information, local storage is used for permanent storage of application data, and session storage is used for temporary storage of data during a session.

3. What is the event loop and how does it work?

Ans: JavaScript is a single-threaded language, meaning it can execute one task at a time. It is also designed to handle asynchronous operations, such as making API requests or running timer-based functions. The event loop has one simple task - to monitor the call stack and callback queue. If the call stack is empty, the event loop will take the first event from the queue and push it onto the call stack, which effectively executes it. The event loop ensures that the execution of a non-blocking task does not block the execution of another task, allowing JavaScript to handle multiple operations simultaneously. This is an important feature for building responsive and efficient web applications that can handle user interactions and external events.
